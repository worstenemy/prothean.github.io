Join优化

Join Access Type

在对查询求值时，MySQL会决定查询的访问类型（Access Type）是什么。下面是一些按从好到坏排序的访问类型：

  • system：system表是一个常量表
  • const：常量表
  • eq_ref：有unique或者主键索引的等值关系
  • ref：有索引的等值关系，索引的值不能为空
  • ref_or_null：有索引的等值关系，索引的值有可能为null
  • range：有索引的范围关系，比如BETWEEN，IN*，>=，LIKE*等等
  • index：有索引的扫描
  • ALL：全表扫描

优化器会使用访问类型来选择driver expression（驱动表达式），考虑如下的查询：

因为indexed_column有更好的访问类型，那么indexed_column更有可能成为驱动表达式，这里只是一个简单的例子，还有很多的例子是一些前面所描述的例外。

成为一个驱动表达式有什么好处呢？考虑一个查询有两条可行的路径：

  • 坏的查询路径：从表中读取每一行（也被称作sequential scan或者table scan）。对每一行，检查indexed_column和unindexed_column是否满足查询条件。
  • 好的查询路径：通过索引查找indexed_column为5的行（也被称作indexed search），对每一行，检查unindexed_column是否满足条件。

index search会比table scan查询的行数更少，也就是更少的访问次数，这也是为什么选择indexed_column为driver更好。
